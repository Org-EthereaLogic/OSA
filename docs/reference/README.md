# docs/reference

Preserved local reference material that may inform OSA decisions but is not canonical for implementation claims.

## What Belongs Here

- Archived external inputs that informed planning or implementation
- Preserved internal notes that are useful to keep but should not steer current decisions
- Framework notes or imported references that should stay separate from the living SDLC suite

## Boundaries

- Do not cite this directory as primary product or architecture truth when `../sdlc/` or `../adr/` disagree.
- Do not store active product, architecture, safety, or release specs here.
- Do not use this directory for prompts that belong in `../prompt/`.
