# docs/adr

Accepted architecture decision records for OSA.

## What Belongs Here

- Decisions that change long-lived implementation constraints, trust boundaries, platform baselines, or product-policy rules
- Accepted ADRs that should remain addressable even after follow-up implementation work

## Current ADRs

- `ADR-0001-offline-first-local-first.md`
- `ADR-0002-grounded-assistant-only.md`
- `ADR-0003-online-knowledge-refresh-with-local-persistence.md`
- `ADR-0004-ios18-minimum-target-with-foundation-models.md`

## Rules

- Add a new ADR when a decision changes architecture or policy in a durable way.
- Do not use this folder for drafts, prompts, or general planning notes.

## Linking Guidance

- From `docs/sdlc/`, link here as `../adr/...`
- From `docs/prompt/enhanced/`, link here as `../../adr/...`
